     Multiple Projects

In a cluster situation you will typically have multiple projects that you are working with. Similarly if you are a consultant, you will likely be working with multiple projects.

OurWord remembers the most recent projects that you have worked with, and places them at the bottom of the Project dropdown menu. Thus instead of opening a project every time you want to look at it, simply look for it in the Project dropdown.

Projects often benefit from sharing some of the same settings, and OurWord facilitates this. The two most typically shared settings are:

  • Multiple projects will often use the same source / front translation. In the New Project wizard, you can specify an existing front translation, rather than inventing a new one for each project.
  • Multiple projects will often use the same Team Settings (which includes the stylesheet, writing systems, and other similar settings.) For any project, you can specify which Team Settings to use through the Team Settings tab on the Tools-Configuration dialog.

When any of these shared settings are changed, the changes are reflected for all projects that make use of them.
