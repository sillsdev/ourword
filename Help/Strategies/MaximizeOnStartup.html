     Maximize on Startup

Most software programs restore the window to the screen coordinates of where it was when it was previously shut down. If it was previously maximized, then it is maximized when started again. If it was previously taking up a small part of the screen, then when started again it continues to have that small size and location.

In working with new computer users, however, we found that window manipulation is one of the more difficult aspects of software to learn. Thus if a window's size has, by mistake, been made too small; the user is often at a loss as how to restore it to a more usable condition.

For this reason, OurWord's default behavior is to always maximize on startup. We figure that for most translators, they will want to work with a maximized window, and thus we should support that behavior.

The advisor can go into the Tools-Configure dialog, under the Options tab, and turn off this automatic "maximize on startup" setting. Having done so, OurWord will then behave in the more normal fashion of starting up in the same location and size as when last shut down.

Updated as of Version 1.0.
