     Locking a book from edits

The Locking feature instructs OurWord to present a book for display, but to not permit editing. Users can add To Do notes, but can not make any other types of changes.

There are several reasons why this feature might be used:

  1. A book has been published, and therefore no further changes are desired; yet notes can be made in case a revision is ever done.
  2. A book has reached the stage where other people are involved who are working on other computers, and it is desirable to prevent changes from being made on more than one computer at the same time (thus avoiding a merge problem.)
  3. This helps guard the integrity of the process. Things cannot be changed on a whim or personal preference, that have already been thoroughly processed with the team advisor, a community check, or a consultant check. At this point any suggested changes to the text now have to go through proper channels and be evaluated against the bigger picture.

A book is locked through a checkbox on the Book Properties dialog (which is accessed from the Configuration Dialog.)

Once a book is locked:

  • The book no longer is displayed with the white background (white is used in OurWord to indicate text that can be edited.)
  • If typing is attempted, a message is displayed to the user informing him that editing is not permitted,
  • Other visual indicators include red text in the navigation dropdowns and the Configuration Dialog, and a "padlock" icon in the view's titlebar.
  • The user can select text, and then create a To Do note for that text.

To unlock a book, simply return to the Book Properties dialog and clear the checkbox.
