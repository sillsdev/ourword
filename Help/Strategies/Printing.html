OurWord Help
     Printing

Printing is accessed through the Print button. (If this button is not visible, use the Setup Features Dialog to turn it on.) This command brings up a dialog, which allows you to set up the print job.

The dialog allows you to set up the folowing:

  • The Printer combo box contains all of the printers that are currently available on your computer or network. Your default printer is shown, but you can use the drop-down arrow to choose a different printer.
  • The Range defines exactly what Scripture you will be printing:
    • Entire Book will print the book that you currently have loaded
    • Current Section will just print the section that you are currently displaying
    • Pages will print the range of pages that you define.
    • Chapters will print the pages that contain the chapters you define
  • Print Watermark will print a light-gray background on each page giving the status of the book, e.g., "Draft" or "Team Revision." This can be helpful in reinforcing to a reader that the printout is not the final version, and thus comments and changes are welcome.
  • Make Substitutions converts angle brackets into quotes.
  • Print Pictures, if unchecked, will save paper by omitting pictures in the text.
  • Line Spacing permits you to choose between Single, 1.5 and Double. With the larger line spacings, it is easier for a person to hand-write comments.

Advanced Print Options

The Configuration Dialog, Team Settings section, allows you to specify other print options:

  • The composition of the footers on odd and even pages. For each footer, there are three parts (left, middle and right), and these parts can be set to any of the following options:
    • Leave Blank
    • Page Number, which appears as, e.g., "- 3 -"
    • Book Stage and Date, which appears as, e.g., "Draft - 6/1/2006"
    • Language, Book Stage and Date, which appears as, e.g., "Navajo - Draft - 6/1/2006"
    • Copyright Notice, which is the string you enter in the Copyright Notice setting. This will be a team-wide notice. You can also define noticies for individual books, by going to the Book Properties Dialog.
    • Scripture Reference, which appears as, e.g., "John 3:16." (This is the first verse number that appears on the page.)
  • The contents of an optional copyright notice.

The font used for the footer is the "Running Header" style, which you can change in the StyleSheet. This may especially be desirable if you use a lengthy copyright notice.

Printing Behavior Notes

The printing implementation currently does a number of things that are normally configurable in a standard word processor. The goals in this implementation were primarily

  1. A feature that would be relatively simple to use, and
  2. Intermediate printouts for the translation process (as opposed to final copy for publication.)

Thus standard, implemented behavior includes:

  • Widow and orphan control - a single line in a paragraph will not be left by itself on a page.
  • Keep-With-Next - Paragraphs such as section headings will not appear on a page unless there is room to start their following paragraph.
  • Drop Caps - Chapter numbers are produced as drop capitals, such that they take up two lines.
  • Footers - the contents of footers are set up in the Configuration dialog and may include, e.g., the page number, the name of the book, the date of the printout.
  • A line of text will not appear on a page unless all footnotes for that line can also appear on that page.
  • Footnote letters begin with 'a' for each page (as opposed to beginning with 'a' for each section as in the screen display.)
  • Pictures are centered; OurWord does not yet support wrap-around text.
  • The watermark is automatically the Status of the book, centered on a 30 degree angle, using the largest font size that will fit in the available space.
  • Page margins, portrait vs landscape, page size, etc., are all whatever is the default for the printer chosen. OurWord does not currently support tweaking of these.

Thee is tremendous complexity in printing; I have a new appreciation for the programming behind software like Microsoft Word. Nevertheless, please feel free to Send feedback as to issues that you find especially limiting.
