OurWord Help
     Automated Hyphenation

OurWord offers a quick-and-dirty means of hyphenation, which admittedly will not be sufficient for some languages. Depending on expressed user interest, future versions may incorporate a more rigorous approach to the problem. For now, though, we hope that a reasonable number of language can benefit.

1. Setup

Hyphenation is set up in the Configuration Dialog, under the Team Settings section, the Writing Systems tab. Because each writing system is potentially different, OurWord permits you to set up a different hyphenation definition for each one.

1. You must first turn on the feature by changing "Turn On Auto-Hyphenation" to "Yes". By default it is set to "No", because automated hyphenation has a high potential to confuse new computer users.

2. At the heart of hyphenation is to specify a CV pattern, where you indicate where the hyphen is permitted.

  • "VC-C" works for many English words, such as "bog-gle", "com-mit", and "air-plane." Unfortunately it fails for words such as "pa-per," thus it is recognized that OurWord will not completely cover all of the hyphenation rules of many languages.
  • "V-C" works for Huichol words, where closed syllables do not exist, thus "pü-ca-heu-xü-ca-cai-ri"

(Please feel free to contact me with your requests on how I might enhance this feature in the future to support your language. I do know that the "right" way is to support the ICU library, and imagine that I will indeed at some point, again, depending on user demand for this feature.)

3. You need to tell OurWord what your consonants are. Because we then know the consonants and the punctuation, we can infer that everything else is a vowel.

4. Finally, the "Min. Split Size" is your way of telling OurWord not to break a word down into parts that are too small. Thus a minimum size of three would prevent "ap-ple", because "ap" would be too small.

2. Behavior

If turned on, hyphens are automatically created, and automatically update as you type or otherwise edit the text. The hyphens appear outside of the white background, indicating that the user cannot type on top of them. Hopefully the user will not be tempted to type in hyphens, because doing so would cause them to be part of the data. If you ever see a hyphen with a white background, it was inserted by the user, not by OurWord.

If a single word becomes too long to fit in a line, and if hyphenation is turned off (or if the hyphen rule does not apply), then OurWord will arbitrarily display a hyphen in order to keep the line from overflowing from view on the screen. Again, these hyphens are not inserted into the data, they merely appear on the screen.

Hyphens are not yet implemented in the Print routine. I know, I know......stay tuned.....and the more voices that request it, the higher this stuff moves up on my To Do list.
